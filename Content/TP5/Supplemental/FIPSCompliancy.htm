FIPS and TurningPoint®

This document outlines FIPS compliance and TurningPoint (version 5.x) in a Microsoft Windows® environment.

TurningPoint can be compliant with FIPS (Federal Information Processing Standard) 140-2. FIPS 140-2 specifies how particular algorithms can be used, and how encryption keys are to be developed and managed.

How is TurningPoint FIPS compliant?

  • During installation, if TurningPoint detects that the computer is FIPS compliant, the software disables any functionality within the application that utilizes the MD5 hash algorithm.

How does TurningPoint determine if the operating system is FIPS compliant?

  • The following queries are performed at runtime:
    • Windows XP/Server 2003
      • HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Lsa\FIPSAlgorithmPolicy
    • Windows Vista and newer
      • HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Lsa\FipsAlgorithmPolicy\Enabled

Which features are disabled in the FIPS compliant version of TurningPoint?

  • ResponseWare (disabled ONLY in TurningPoint 5.0 and 5.1; ResponseWare in TurningPoint 5.2 and higher does not require the MD5 hash algorithm)
    • A check is performed when calling Login to see if the environment is enforcing FIPS compliance
  • Moodle and Sakai integrations
    • A <FIPSCompliant> node was added to the lms_properties.xml to denote which systems are not FIPS compliant
  • HID receiver piece that prevents compliance

Does this product inter-operate between two services?

  • Yes. Turning Technologies utilizes web services to communicate to the following: automatic updates (turned off by default in TurningPoint), ResponseWare and K12 Standards (all hosted by Turning Technologies) and third party Learning Management Systems (LMS). ResponseWare and LMS services are turned off by default when in a FIPS compliant environment.
