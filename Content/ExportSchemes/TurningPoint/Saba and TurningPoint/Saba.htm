TurningPoint 5® TurningPoint® Cloud and Saba®

This document outlines the steps for using TurningPoint Cloud with the Saba Learning Management System (LMS).

How does the integration work?

A participant roster (report) is downloaded from Saba and imported as a CSV into TurningPoint Cloud. The participant list is loaded before you begin your polling session. Once the session is run and the results are saved, it can be opened in Results Manager. After viewing results and making any necessary edits, the session can be imported into Saba.

Getting Started - For System Administrators

The following tasks MUST be completed for a participant roster (report) to properly export as a CSV to TurningPoint Cloud.

Placing the Roster Report File (RPT) into the Report Repository

  1. On the machine where the Report Saver is running and relates to the Saba environment, copy the TTech.rpt file into the directory where the other managed Saba reports are stored. This will more than likely be a directory path similar to <SabaWeb>\web\CrWeb.
    1. Note that the directory structure will be configured in the Crystal Reports Embedded report engine definition accessible to a user with System Administration capabilities in the Saba UI.
    2. The configuration of the Embedded report engine is accessible through the following click-path in Saba: System Administration > Reports tab > Report Engine left-menu item > Engines left-menu item. Click the link for the name of the Crystal Report Server XI Embedded. The directory path where the RPT files are kept is the Report (.rpt) File Location attribute.

Creating the Report Tool

Creating the Report Tool only needs to be done ONCE per Saba environment!

  1. Login to Saba as a user with System Administration capabilities.
  2. Select System Administration from the Go To drop-down menu.
  3. Select the Reports tab.
  4. Click Report Engine from the left menu.
  5. Click Managed Reports.
  6. Click the New Managed Reports link.
  7. Enter the following attributes on the New Managed Reports page:
    • Name - Anything that makes the report stand out in a list of reports (i.e, Turning Tech Roster)
    • Enter a description, if desired
    • Engine - Enter cr and select Crystal Report Server XI Embedded from the drop-down menu
    • ID - Enter TTech.rpt
    • Attach to - Enter sys and select System. This makes the report available below the Reports tab.
    • Domain - Ensure the domain is visible to user(s) who will run the report
  8. Click Save.
  9. Enter the following attributes on the Edit page:
    • Report Parameters - Click the link to Add Parameter
    • Description - Enter Enter Offering Number(ID)
    • Parameter Type - Select Text(10)
    • Parameter Sequence - Enter 1
    • User Input Required - Select Yes from the drop-down menu
    • Click Save. The parameter details entered above are listed on the Report details page.
    • Menu Roles - Click the Add Menu Role link. Select the Learning Administration menu role and click the Select button.
    • Security Roles - Click the Add Security Roles link. Search for and select the following security roles (a standard set for running reports).
    • Super User
    • TenantAdministrator
    • Report privileges in world domain
  10. Click Save. The report is now accessible.

Before Class - For Learners or Trainers

Running the Turning Technologies Roster Report

  1. Login to Saba as a user with Learning or Training Administration capabilities.
  2. Select Learning Administration from the Go To drop-down menu.
  3. Search for the course offering by Title, Start Date, etc.

    Note the ID value shown in the search results and on the Details page.

  4. Select the Reports tab.
  5. Search for the Turning Tech Roster report. (The report may be named something other than Turning Tech Roster.)
  6. In the results list, mouse over the Actions link for the desired report and select Execute.
  7. Enter the offering ID noted above into the entry box.
  8. Select Generate Report.
  9. In the report viewer, click the icon to the left end of the line that displays the group tree above the report results. This will export the report.
  10. In the export dialog box, select the file format Separate Values (CSV) and click Export.
  11. Save the file to a specified location.

Importing the CSV File into TurningPoint Cloud

  1. Open TurningPoint and select the Manage tab. Open TurningPoint Cloud and sign in to your Turning Account.
  2. Select the Manage tab.
  3. Click the Participant List drop-down menu and select Import.

    The Import Participant List window is displayed.

  4. Select the radio button labeled File from Integration.

  5. Select Saba from the Select a Format drop-down menu and click Choose File...
  6. Select the file that was saved in the previous section and click Open.

    The Saba file is converted to a TurningPoint Cloud participant list and can be found under the Manage tab.

  7. Select the Saba participant list from the left panel.
  8. Click Edit Participants in the bottom right corner.
  9. Click Column and select Add.
  10. Enter ID and click OK.
  11. Select the blank field under ID and enter 1. Select the field under 1 and enter 2. Continue this process until you have the same amount of ID numbers as participants.
  12. Click Save and Close.

Uploading Grades to Saba

  1. Login to Saba.
  2. Select System Administration from the Go To drop-down menu.
  3. Click the Data Import tab.
  4. Select Ad Hoc Transcript from the Object Name drop-down menu.
  5. Select Delimited as the Import Type.
  6. Enter a comma (,) in the delimiter field.
  7. Click Browse to locate and open the saved file.

  8. Click Prepare.
  9. Click Import after receiving the message, The data in the specified file has been staged.
  10. Click Status to view the results of the import.
