Using Insight 360 Cloud with Reimaging

Many organizations use enterprise solutions that revert PCs back to a predefined state, upon the PC reboot. If your organization employs such a solution, you will want to ensure that the various pieces of Insight 360 Cloud are configured correctly, for optimal experience.

Below you will find a list of files and directories, used by Insight 360 Cloud, that should not be modified, as well as a list of minimum user permissions needed to run the software.

  • The paths listed are the default paths. If a custom install was performed, these paths may vary.
  • Computers must be unlocked, or “thawed,” for any permanent changes to take effect.
  • The minimum permissions and ability to reimage on reboot are with the assumption that Insight 360 Cloud has been installed, registered, and activated before being placed minimum privileges (or reimage) on reboot setting.
