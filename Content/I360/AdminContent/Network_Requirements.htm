Network Requirements

Instructor’s Computer

  • Internet connection, for items such as registration and reporting
  • Access to http://insight360services.einstruction.com, http://reports360.einstruction.com and http://insight360.einstruction.com
  • Whitelist Insight 360 Cloud on firewalls, including local machines, such as Windows Firewall
  • Insight 360 Cloud optionally uses IGMP multicast to make joining from teacher and participant mobile devices easier (this is called service discovery). Utilizing IGMP allows the instructors and students to select their class and automatically connect (compared to using an IP Address to connect to the classroom). For this to work, IGMP and Multicast must be enabled on the wireless network. Insight 360 Cloud uses the IGMP address of 239.255.107.13.
  • The following TCP ports should be allowed to receive connections:
    • 65007 - To accept connections from teacher mobile device
    • 65009 - To accept connections from participant mobile devices for Premium
    • 65008 - To accept connections from participant voting automated Pulse devices
  • Also, UDP port 31009 must be open to allow receipt of Multicast UDP messages if service discovery is desired.
  • If your deployment includes third party integrations via Insight 360 Cloud’s HTTP API, then TCP port 64250 must also be open on the instructor’s computer to receive HTTP requests from the third party application.

Insight 360 Teacher App

  • Teacher mobile device should be on the same local area network as the instructor’s computer
  • Local high-speed Wi-Fi

Insight 360 Cloud Student App

  • Participant mobile devices should be on the same local area network as the instructor’s computer
  • Local high-speed Wi-Fi
