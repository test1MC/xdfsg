Insight 360 Cloud Custom Settings

This feature is available with Insight 360 Cloud v2.3.1+

Insight 360 Cloud will look for an XML file called CustomSettings in the top-level data directory to populate settings on the Preferences, Registration/Activation and Mobile Devices tabs. If the file does not exist, the program defaults will be used. If the file exists, and the contents parsed correctly, the normal defaults will be overwritten by those in the CustomSettings.xml file. This feature exists to help IT departments in large deployments of Insight 360 Cloud to be able to change Insight 360 Cloud's default settings and/or to help users complete the registration and activation process.

The CustomSettings.xml file must be in the Insight360 local data folder (where the log files are stored), NOT in the data directory itself:

Platform

Path Name

Windows

C:\{User}\AppData\Local\Insight360\CustomSettings.xml

Mac OS

Users > [username] > Library > Application Support > Insight360 > CustomSettings.xml

The Custom Settings feature does NOT modify the registration and activation process in anyway - it merely allows the registration fields to be pre-populated by the CustomSettings file. The CustomSettings file can be unique to an individual machine/user or it can be generalized for a set of users.

Other notes:

If a tag in the CustomSettings.xml file is not formatted correctly, i.e., it is not understood by the software, it will be ignored and the last saved or default value will be used instead.

Preferences and Mobile Devices settings tabs are loaded upon Insight 360 Cloud Director restart. See the flowchart below for more detail.

The Registration/Activation settings tab is loaded whenever the tab is loaded. If the tab is already loaded (e.g., visible to the user) when a custom settings value is changed, the new value will NOT be read until the tab is reloaded (click on another settings tab and click back on the Registration/Activation tab). See the flowchart for more detail.

The Registration and Activation processed are not changed by this feature. This feature only affects the values of the fields on the Registration/Activation tab. Changing a field value will not change the registration record unless the registration is submitted. Changing the SRC field value will not change the license unless the license is activated. Re-activating a license will not affect the license terms in any way.
