Changing the Default Data Directory Location

If using a custom path, choose the location carefully. If it is a network location or shared location that other users have access to, there is potential for data corruption. If the custom data path cannot be found, read from, or written to, then the application will revert to the default data path.

How to change the default data directory location...

  1. If Insight 360 Cloud is open, close the program and terminate the Insight 360 Cloud Director by pressing Crtl+Shift+Q.
  2. Open SteamPunkSettings.xml in a text editor. The file can be found in the following locations:
    • PC – C:\Program Files\eInstruction\Insight 360 Cloud\Tools\Configuration\SteamPunkSettings.xml
    • Mac – Applications > eInstruction > Insight 360 Cloud > Tools > Insight 360 Cloud Director.app > Resources > Configuration >
      SteamPunkSettings.xml
  3. Within the DesktopHostSettings tag, add the CustomDataPath tag as shown in the example below (in red):

    <?xml version=”1.0” encoding=”utf-8” ?>

    <DesktopHostSettings xmlns:xsi=”http://www.w3.org/2001/XMLSchema-instance” xmlns:xsd=”http://www.w3.org/2001/XMLSchema”>

    <EInstructionServiceUri>http://insight360services.einstruction.com/
    StarfishCloudServices</EInstructionServiceUri>

    <WebClickerUri>http://insight360.einstruction.com</WebClickerUri>

    <GenerateTestData>false</GenerateTestData>

    <SimulateSessionVoting>false</SimulateSessionVoting>

    <SRSReceiver>true</SRSReceiver>

    <CustomDataPath>P:\Insight 360 Cloud\</CustomDataPath>

    </DesktopHostSettings>

    Replace “P:\Insight 360 Cloud\” with the actual desired location. Do not change any other parameters.

  4. Open Insight 360 Cloud. On startup, a test file (TestWrite.dat) and an Insight 360 Cloud folder with a new database will be created in the custom path location.

    Use of a network location or drive for a custom data path should only be considered if:

    • Network has ample bandwidth to support the frequent read/writes to the custom data path.
    • Custom data path is not shared by multiple machines. This path must be unique per registered user/computer.
