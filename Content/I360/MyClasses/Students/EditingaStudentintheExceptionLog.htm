Editing a Student in the Exception Log

If there are errors during the student import process, the affected students are moved to the Student Import Exception Log. The Error column provides a brief explanation of the error.

The most common exception is a duplicate Student ID.

If a student has not been imported into the Insight 360 Cloud database, the Student Imported column displays No. Student information, such as the Student ID, can be edited.

If a student has been imported into the Insight 360 Cloud database the Student Imported column displays Yes. Student information cannot be edited.

  1. To edit students in the exception log:
    1. In the exception log, double-click the item to be edited.
    2. Enter the new information and press Enter.
  2. To import edited students into the Insight 360 Cloud database:
    1. Select the box or boxes that correspond to the student.
    2. Click Import Selected Students.

    This action occurs immediately and cannot be undone.

  3. To remove students from the exception log:
    1. Select the box or boxes that correspond to the student.
    2. Click Remove Selected Exceptions.

    This action occurs immediately and cannot be undone.
