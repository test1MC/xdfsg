Sample Question Bank

This section includes a sample question bank with the following question types: true/false, multiple choice, matching, completion, and short answer. As you can see the sample includes some of the optional question information tags and corresponding information. A narrative linked to two multiple choice questions is also shown in the example. Refer to the previous sections as you review the sample bank. Use this sample as a template when you begin writing questions using your word processor.

As you review the sample question bank, look for these items:

  • Question bank title on first line of sample bank
  • Question type headings have been added (Multiple Choice, True/False, Completion, etc…)
  • Metadata Tags such as Difficulty (DIF:) and reference (REF:) include a colon
  • Narrative begin/end tags with a narrative name
  • Multiple choice formatting (a., b., c., and d.)
  • Exclamation point in multiple choice to lock that choice(s) so that they do not scramble (Ex. d! All of the above)
  • Use of a table, not tabs for the narrative
  • Minimal formatting (indents, tabs, etc.)

Using the ExamView Import Utility to Import your RTF document

The ExamView Import Utility is located in the ExamView folder (evimport.exe). You can place the RTF files in the Banks folder, also located in the ExamView folder, or create a folder on your desktop. Double click evimport.exe to open the ExamView Import Utility or you can create a shortcut for your desktop. Click Next.

  1. Step 1

    1. A dialog box appears containing two boxes. In the top box search for the files to be converted. Once you have selected the RTF documents by using either the Select or Select All button, the banks you would like to convert are displayed in the lower box.

      Three folder icons are displayed in this dialog box.

      • Top folder: Navigates to the top level of the ExamView folder.
      • Middle folder: Allows you to navigate to the folder housing your RTF files if they are not located in the ExamView Assessment Suite banks folder.
      • Bottom folder: Allows you to navigate to another folder if you want to store your new ExamView Assessment Suite question bank in a location other than the ExamView Assessment Suite banks folder.
    2. Click Next.

  2. Step 2

    1. This screen displays formatting preferences.
    2. The default number of columns for Multiple Choice questions and Matching questions is 2 columns. If necessary, you can adjust the number of columns in the Question Bank Editor (Question | Adjust Choices/Columns) after the import is complete.
    3. Show options is also in this screen. Verify that all Show options are selected. If needed, you can change the Show options in the Question Bank Editor (Edit | Preferences | Layout) after the import is complete.
    4. Click Next.

  3. Step 3

    1. If you have not already set your fonts, this next screen allows you to do so.
    2. Click the Question Bank Fonts to view a drop-down list of the question bank elements that you can change.
    3. Select New Question and click Set.

      The Default is set to Times New Roman, Regular, 11 pt.

      Setting the Question Bank Fonts affects only new questions that are added to the question bank once the import is complete; these settings do not affect the questions from the RTF files. Please note: all fields would need to be changed individually on this screen. In the “Question Bank Fonts” section, there is a drop down box that displays the name for each section. Once you click on the section, the font type, size and special formatting (“Bold”/”Italic”) are displayed on the screen.

    4. Click Next.
    5. Verify that Question Number style is set to the default 1.
    6. Verify that Answer Choice style is set to the default a.
    7. Verify that Show Choices for True/False and Yes/No is deselected.
    8. Verify that Use Columns is selected.
    9. Verify that Choice Sequence is set to the default abcde.
    10. The above items can be changed if necessary in the Question Bank Editor (Edit | Preferences | Style) after the import is complete.
    11. Click Next.

  4. Step 4

    1. In this step the keywords on the right side of the dialog box must match exactly to the keyword that you used in your RTF file. If they do not match you need to change the keyword, based on the keyword list that should now be showing on your screen.

      In the RTF file the answer to a question may be labeled as “Answer:”. The EVImport Utility will be looking for “ANS:”. For the import to work the keyword will need to be changed to “Answer:”.

    2. Use the scroll bar to check every keyword that you are using to verify that everything matches. Once you have verified all information click Next.

  5. Step 5

    1. If you need to make any changes to the information that you supplied, use the Back button and make changes. If there are no changes, click Finish.
    2. Once Finish is clicked, a new document appears on-screen. This displays the status of the import and whether or not problems occurred. If you need a record of the import report, save the output from EVImport to your project folder.
    3. If you are given error messages, it is recommended to go back to your RTF documents and make the appropriate changes. If you receive no error messages, the question banks appear in the folder that you specified in Step 1.

Once questions are in BNK format, please review to ensure all questions and special formatting have imported correctly.
