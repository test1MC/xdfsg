Using Artwork on Tests

Artwork, whether a simple graph or a complex illustration, can enhance any test. Please note that the requirements for each project are different. If you are not sure how to address your specific needs, please contact Turning Technologies before starting your project. If you prefer, you can use placeholders instead of the actual art in the questions (e.g., INSERT FIGURE 1-2 HERE.) and provide the original art to Turning Technologies. Use file names (e.g., figure 01- 2.bmp) for the art so that they correspond to the placeholders in the question banks. Use the following considerations when inserting art into a question or narrative.

File Formats

ExamView converts all art to bitmaps (BMP) once inserted into question banks. Bitmaps do not contain compression and will display better inside of ExamView than other formats. Therefore, it is recommended to save all files, whether black and white, color or grayscale, as bitmaps. If using a file format other than bitmap, be aware that the image quality may not be as crisp as the original.

Make sure that each image (text, details, etc.) is legible

Images used in ExamView Test Generator must be legible on paper and on-screen. Check the legibility of the text and other elements to verify that a student can answer the associated question. Art may display differently on different platforms (Mac/PC) and/or operating systems.

Art used in a textbook or in a test booklet may appear crisp and clear when printed at 1200 dpi (dots per inch), but be completely unreadable when displayed on a computer screen. For example, a detailed map may include minute details that cannot be seen on a monitor. In this instance,contact your Turning Technologies representative to discuss the best course of action.

Bit Depth

Saving art as monochrome (or 1-bit) bitmaps produces crisp line art with the smallest file size. It is recommended to convert art to monochrome bitmaps whenever possible. If gray tones or colors are necessary, files can be saved in 4-, 8-, or 24- bit depth. Because of possible conversion problems going from PC to Mac, do not save images in 16-bit depth. Note that the larger the image size, the longer it will take to open the bank files.

Use black and white images whenever possible

Black & white (1-bit monochrome) images are smaller than images created in any other format. Often, simple illustrations or graphs are saved in a 256-color format even though the only colors in the image are black and white. There may be other times where only a small amount of gray or another color is used but this color is not necessary. Simply reducing the number of colors will dramatically reduce the size of the image.

If the image does contain only black and white pixels but has been saved in a 256-color format or as a 24-bit image, you can use almost any paint program to reduce the colors to black & white. If you need to change an image that has colors or grays to black & white, you probably need to use a more sophisticated image-processing program (Adobe PhotoShop).

When you create black and white art, it should be created at a larger size and then scaled when you insert it into a question. This results in an image that is higher in resolution for better print output. As a general rule, you should scale black and white images at fifty percent.

Never use Metafile 24-bit or 16-bit images

Images created in 24-bit or 16-bit mode (a very high resolution resulting in a large file size) are not appropriate for ExamView® Test Generator. These modes are required to produce 4-color books, but will cause unnecessary memory problems and performance issues for programs designed to display on the screen. Even programs such as Quark® or Photoshop® can take several minutes just to print or display a large 24-bit image. If you have 16-bit or 24-bit images inserted into your questions, simply click on the image you want to change. A border with handles (small squares) appears around the image. Choose Format Picture from the Format menu. From the Size tab, enable the Lock aspect ratio check box if you want the program to automatically adjust the image in proportion to a change in either width or height. Set the scaling or the width/height to change the size of an image. Set the baseline. From the Picture tab, you can choose the Picture type that requires the least amount of memory while retaining the desired picture quality. Use the Crop from feature if you want to cut any sections off the edges of the picture.

Never use enhanced metafiles

Enhanced metafiles DO NOT display on Mac computers. These images can be converted to 24-bit or 256 bitmap and will display on both Mac and PC computers.

Never scale a color or grayscale (16-color or 256-color) image

You should not scale color images in ExamView® Test Generator. Doing so will blur the image on screen. If there is text in the image, it may make the text unreadable. Note: Any scaling should be done in a program such as Photoshop® before inserting the image into ExamView® Test Generator.

Scale and Dimensions

Only monochrome bitmaps can be scaled down after insertion into ExamView PC Banks without possible loss of image quality. Because scaling improves the image clarity, it is recommended to save monochrome bitmaps at 200% to 400% of the final size at 96 dpi.

If the final test is to be printed and you want the art to be 3 inches wide, save the monochrome art between 576 and 1152 pixels wide and scale to the proper size.

Scaling grayscale or color images saved in 4-, 8-, or 24-bit depth can cause scan lines to drop out on PC displays, risking the loss of critical information.

For PC banks, save grayscale and color images at 100% of final size at 96 dpi, not to exceed 528 pixels wide.

Color images must not exceed 5.5” (width) by 6” (height).

The displayed dimensions of black and white or color images should not exceed the entry area size provided in ExamView Test Generator.

A screen capture from a program may be up to 10” wide. To reduce the physical size, crop the image to show only the necessary parts, or scale it (in PhotoShop) if, by scaling it, the image retains all of the relevant details.

Because instructors can use the two column test format, consider sizing art no larger than 2.75” (width) by 3” (height). This size will allow art to be formatted into two columns.

Artwork that needs to remain large for readability (maps, graphs, etc.) may still be used with the two column format feature. In such instances, the art will remain full size and the formatting for questions attached to that piece of art will revert to one column.

Mac vs. PC Resolution

Display resolutions vary; PC is typically 96 dpi while Mac is 72 dpi. This means images created for PC question banks will be automatically scaled down when converted to Mac.

If a monochrome bitmap was scaled at 50% in PC, it will be scaled at 33% on Mac. 4-, 8-, or 24-bit images inserted into Mac question banks should be scaled at 75%, so that they will be automatically scaled at 100% when they are converted to PC.

While it is important never to scale 4-, 8-, or 24-bit images inside of PC banks, images in Mac question banks can be scaled without affecting quality.

Section 508 Compliance of the Rehabilitation Act

It is necessary to use consistent guidelines for the alternate text to be readable by most screen readers. To establish the preliminary guidelines for an ExamView Alternate Text project, we referenced the best practices for alternate text promoted by the Center for Persons with Disabilities at Utah State University on http://webaim.org/techniques/alttext. Other websites offer similar guidelines.

Alternate text attributes typically include the following:

  • Be accurate and equivalent in presenting the same content and function as presented by an image.
  • Be succinct. Present the correct content and function of the image as succinctly as is appropriate. Typically no more than a few words are necessary, occasionally a short sentence or two may be appropriate.
  • Do not provide the same information as text within the context of the image.
  • Do not use the phrases "image of ..." or "graphic of ..." to describe the image. It is usually apparent to the user that it is an image. And if the image is conveying content, it is typically not necessary that the user know that it is an image that is conveying the content, as opposed to text.

Include alternate text for the following:

  • All images
  • All ExamView Equation Editor content
  • Entire equations and expressions

Key the following content in ExamView’s Equation Editor:

  • Any math content (e.g., equations, fractions, radicals, etc.) that imports into ExamView as an image
  • Any of the following symbols that appear in an equation or expression: -, ≠, ≤, ≥, », p, Ð, ×, ٪, <, >, radicals, exponents, among others
  • Names of all lines, line segments, polygons, etc. For the most part, avoid using vowels (i.e., A, E, I, O, U, and Y). If used, enter one horizontal space between the letters of the name in the alternate text.

Quick Reference Recommendations

  • Save art files as bitmaps. (BMP)
  • Save art as monochrome bitmaps whenever possible.
  • Save monochrome bitmaps 2 to 4 times larger than final size.
  • Scale down monochrome bitmaps inside of PC ExamView banks for best results.
  • Save color and grayscale images as 4-, 8-bit, or 24- bitmaps.
  • Do not save images in 16-bit depth.
  • Do not use enhanced metafiles.
  • Do not scale grayscale or color images inside of ExamView Assessment Suite PC banks
  • Maximum width for grayscale and color images is 528 pixels.

Inserting a Picture into a Question or Narrative

  1. Create a question or narrative.
  2. Position the cursor where you want to insert the art.
  3. Click Insert from the menu bar and select Picture.
  4. Locate and select the picture you want to insert and click Open.

    See the online help for the kinds of pictures you may use (e.g., .bmp, .jpg, etc.).
