Computerized Testing Basics

Before you begin writing questions for use with the ExamView Assessment Suite software, make sure that you are comfortable using Microsoft Word. The next step is to understand some of the issues you will face when you write questions for use with test generator software. After you learn about ExamView Assessment Suite, the next step is to understand some of the issues you will face when you write questions for use with test generator software. Given the nature of paper and pencil tests, there are few (if any) limitations. When you provide questions in a program such as ExamView Assessment Suite, you must keep some basic rules in mind.

Review the following information in preparation for writing questions.

Question Bank Title

You must include a title on the first line of a question bank document. This title can be mixed case (upper and lowercase) or uppercase. You are limited to 79 characters for the question bank title. If the number of characters on the first line of your document file is more than 79, the title will be cut off. It is suggested that you use this line for the Chapter number and Title (Chapter 1: Getting Started).

Text on the second line that is not a question type header will be used as the bank subtitle. You are limited to 79 characters for the subtitle, as well. The subtitle field is only visible on the question bank editor side and will not appear on a test.

Question Type Order

Questions within a RTF file must be grouped by question type. What this means is that all of the true/false questions must be in the same group under the heading "True/False," all of the multiple choice questions must be in the same group under the heading "Multiple Choice," and so on. Once imported, version 7.50 and higher question bank (BNK) files support mixed question types.

Question Type Instructions

The ExamView Assessment Suite software automatically prints standard, student instructions for the objective-based question types (e.g., multiple choice) when question types are not mixed. Therefore, you are not required to provide any instructions for these types of questions.

ExamView Assessment Suite version 7.5 and later allows question types to be mixed. It is important to note that default instructions do not appear when question types are mixed. If using the mixed question type feature, instructions should be written as a narrative. For more information, see Narratives.

For the open-ended question types (e.g., Essay, Problem, Case, etc.), it is recommended to provide the instructions either as part of the actual question or in a narrative. For example, a problem question may begin as follows: Based on the information provided, calculate the result and explain your answer…

Default student instructions provided by ExamView Assessment Suite:

True/False: Indicate whether the sentence or statement is true or false.
Modified T/F: Indicate whether the sentence or statement is true or false. If false, change the identified word or phrase to make the sentence or statement true.
Multiple Choice: Identify the letter of the choice that best completes the statement or answers the question.
Multiple Response: Identify one or more choices that best complete the statement or answer the question.
Yes/No: Indicate whether you agree with the sentence or statement.
Completion: Complete each sentence or statement.
Matching: (none)

You must insert an instruction line for Matching type questions. Since all default instructions are italicized, you may wish to italicize matching instruction lines.

Numeric Response, Matching, Short Answer, Case, Essay, Problem, & Other: (none)

Question Numbers

Questions must be numbered consecutively either throughout the question bank (TF 1-15, MC 16-30, SA 31-50), or within a section (TF 1-15, MC 1-15, SA 1-20). Question numbers must be the first visible characters on a line, must not be in a table, and must be followed by a period.

Modified True/False Concerns

With ExamView Test Generator you can use a derivative of the True/False question type—Modified True/False. This question type requires a student to provide extra information to make a false statement correct. If you are mixing questions in the question bank file, consider creating a narrative to clarify that these questions are modified true/false.

You must underline or italicize a word or phrase in each question. Provide an answer line (at least 20 underscores in length) for the student to enter a response for “false” statements. For questions that have “false” for the answer, you must provide the answer “F” and a “corrected” word or phrase in the answer area. This correction should not be an entirely new sentence. The number of choices in a multiple choice question is limited to a maximum of eight (a. – h.). Only one answer may be specified. Choices may include the following: All of the above, None of the above, or Both A and B, etc.

Multiple Response Concerns

When creating a multiple response question, include the phrase (Select all that apply.) after the question. Since default directions do not appear if question types are mixed, adding (Select all that apply.) will provide direction for the student that questions may have multiple correct responses.

Matching Concerns

A matching group is limited to 26 choices (a. – z.) and 26 questions per group. However, the number of choices is not required to equal the number of questions. For example, you could have four choices and ten questions (or vise versa). Divide a matching group into two or more groups if there are more than 26 choices. A matching group can be formatted using standard text choices or a picture as shown below. The preference is to italicize the instructions. If adding PTS: to the matching questions, please be aware that if your matching questions exceed 8 per set, the PTS values will not import correctly. These must be manually corrected once the questions have been imported into ExamView.

Blackboard limits the number of Matching questions per set to 20. If your question bank content will be imported into a Learning Management System, limit your Matching questions to 20 questions per set.

Completion (Fill-in-the-Blank) Concerns

When you create a completion question, provide a blank (write-on line) in the question for the student to write his/her answer. The length of the line should provide enough space for the student to write a response (e.g., use a minimum of 20 underscores, increasing by increments of 5 underscores as needed to accommodate the answer). Use the shifted, hyphen key to create the write-on line. Do not create the write- on line using “underlined” spaces.

You can provide more than one possible correct answer for a completion question. Place each answer on a line by itself. For online tests, the software will accept any of the answers. If you use one or more write-on lines for a question, put all of the answers on the same line separated by commas.

Questions Must Stand Alone

Every question must stand alone unless it is specifically linked to a narrative. For example, do not write a question that refers to another question. Also, do not require that questions be presented in a particular order. Directions need to be in a narrative since question order can be mixed. For more information about narratives, see To Create and Use Narratives.

The sample question given below is incorrect since it refers to information in another question. When an instructor creates a test, he/she may not select all of the questions. Also, the instructor may change the order of the questions.

INCORRECT

1. XYZ Company started operations with $40,000 in capital. Expenses are expected to reach $60,000 by the end of the year. Expected revenues are $2,500 per month. What is the expected profit or loss?

ANS: The expected loss is $30,000.

2. Based on the information in the previous question, what level of revenue is required to break- even?

ANS: The break-even revenue point is $60,000

Narratives

When more than one question depends on the same information, you should use a narrative—a table, descriptive passage, special instruction, picture, or other common block of information. Directions need to be in a Narrative since question order can be mixed.

All questions linked to the same narrative (from each question type) print together as a group on a test, but the instructor is not required to select all of the questions. You can attach a narrative to any question type except matching. Matching questions cannot be linked to a narrative because the choices already act as a narrative. For figures and tables, label the information and refer to it by that label in a question. See the examples presented if you plan to use narratives.

To identify a narrative, you must use the NARRBEGIN: and NARREND tags. Everything between these tags will be inserted into a narrative and given the name supplied as part of the NARRBEGIN: tag. For each question you want to link to a narrative, include a NAR: tag and the narrative name below the ANS: tag. See the examples that follow.

NARRBEGIN: XYZ Company

XYZ Company started operations with $40,000 in capital. Expenses are expected to reach $60,000 by the end of the year. Expected revenues are $2,500 per month.

NARREND

ANS: The expected loss is $30,000.

NAR: XYZ Company

2. Based on the information shown above, what level of revenue is required by the XYZ Corporation to break-even?

Question Length

Although there is no physical question length limit, it is recommended not to exceed a full page length for question and answers. Longer questions may cause pagination problems when printing tests. If you are writing questions to be used on an online test, it is a good idea (when possible) to limit question length so that the question can be read on screen without scrolling.

Question Bank Limit

The ExamView Assessment Suite software stores questions in files referred to as question banks. Each question bank contains everything related to the questions in that particular bank just as a Microsoft® Word file includes all of the items contained in a document. All of the images, question information, equations, etc. for a question bank are stored in one file.

A question bank (chapter, test or section) is limited to 250 questions. However, the number of question banks is unlimited.

Question Bank File Names

When you save a question bank, use a maximum of 27 characters (not including the extension). This is a requirement whenever the question banks will be converted to the Mac since its limit is 31 characters for file names.

Use a consistent file naming convention (e.g., Chapter 01.bnk, Lesson 01.bnk, etc.). If you choose, you can add other identifying information in the file name such as an ISBN number (e.g., Chapter 01 0-3345-1.bnk).

Special Symbols and Fonts

The default font for the ExamView software is Times New Roman, 11pt (PC) and Times, 12 pt (Mac). You may also use Courier New and Arial (PC), or Courier and Geneva (Mac) and Wingdings. If you need special characters, you may use the Symbol font or Wingdings. For example, Times New Roman Greek characters (a,b,p…) must be changed to Symbol font in order for them to be read on Win/Mac operating systems and in some Learning Management System (LMS) applications.

If you use other special fonts or characters (other than Symbol or Wingdings), the instructor using the question bank may not have the font you used and, therefore, will not be able to display/print those character(s). In addition, if the banks are to be used for LMSs, (Blackboard, Angel, etc…) those special fonts or characters may not be supported.

Font guidelines are also important since instructors may choose to publish tests to the Internet. By using the default fonts, students who download the test from the Internet should have the required fonts to properly display the test.

Use only plain text in the metadata fields. Do not include any special formatting (e.g., bold, italic, sub/superscripting, etc.) and/or special characters (e.g., SY∞). If you need to use any special features, use the Rationale field.

Tables vs. Tabs

If at all possible, tabs should be avoided, in questions, answers and narratives. Avoid tabs to keep questions "HTML ready" so that tests can be published to the internet. If questions (or narratives) need to be formatted with tabs, you should convert the text (and tabs) to table cells. The table does not need to have borders, and you can use justification within cells to mimic the appearance of decimals or right tabs. Do not use decimal alignment in table cells. In this instance, you would set the columns to right justification.

Points

Use the Points preferences to assign the default point values for the different question types. Points can be printed on a test or used in an online test to give more control over assigning credit for incorrect/correct answers. For multiple response questions, where there may be more than one correct response, partial credit can be assigned if a student does not identify all of the correct responses. The default point value is 1.

Student Feedback

ExamView Test Generator can generate paper and online tests. For tests delivered online, instructors may choose to display feedback for incorrect responses. Practice tests can be set up to display the reference and rationale information. For multiple choice questions, you can set up feedback for each response. For true/false, multiple response, and yes/no questions, you can set up feedback based on whether the response is correct or incorrect. Depending on your testing objectives, you may want to consider including this information. (See the ExamView Test Generator online help for detailed instructions.)

Rationale

For multiple choice, multiple response, true/false, and yes/no question types, you can enter a rationale (or explanation) for the answer. If, for example, you create a multiple choice question that involves a calculation, you could enter an explanation of the calculation in the Rationale entry area. This provides more information if you need to explain to your students why a particular answer choice is correct.

The rationale prints on the answer key and is intended for instructor use. You can, however, show the rationale to students who take a study guide (practice test) online.

The rationale entry must be placed on the line directly after the ANS: entry in order for it to import to the ExamView question bank.

Formatting

As a general rule of thumb, keep the formatting to a minimum when you prepare a question bank in Microsoft® Word. For example, you do not have to set up indents for multiple choice questions. You can let the text wrap under the choices.

Artwork

If you plan to use art in any questions, consider size, clarity, and resolution. See Using Artwork on Tests for a detailed discussion of the issues pertaining to this matter.

Equations

Equations that you enter with Microsoft Word equation editor must be re-keyed using the ExamView Test Generator software. For best results, you should use ExamView Test Generator to enter question banks that require equations.
