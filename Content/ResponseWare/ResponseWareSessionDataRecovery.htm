ResponseWare Session Data Recovery

Use the data recovery tool to recover data from a ResponseWare session file containing partial data.

THE FOLLOWING IS FOR INTERNAL USE ONLY. DO NOT DISTRIBUTE TO EXTERNAL PARTIES.

 

  • The TurningPoint session file must have been run at least the day before and exported from the Manage tab.

    The Data Recovery Tool searches the backup database that updates at midnight each day.

    If the session file is not exported from the Manage tab, you will not be able to access it in the Recovery Tool.

  • The password associated with the session file is required to access the data.
  • The session ID or presenter email is required to complete the search.

Go to the Manage tab. Select the session, click the Session drop-down menu and select Export. Select an export destination and click Save. You will be prompted for a password for the session file. We recommend setting the password as "password" (all lowercase). If a different password is used, include the password when forwarding the session file.

  1. Open the ResponseWare Session Data Recovery Tool.

  2. Click Load Session.
  3. Select the TPZX file and click Open. If applicable, enter the password associated with the session file.

    If "password" is used to export the session file, the Recovery Tool automatically loads the session.

    The current session data contained in the session file appears in the bottom left panel.

  4. Click Load Recovery.
  5. Enter the Session ID. Alternatively, enter the presenter's full email address.

    The Session ID is case sensitive.

  6. Click Search.
  7. Select the session and click Recover.

    The available sessions display your local timestamp, which may differ from the loaded session displaying the presenter's timestamp.

    The recovered session data appears in the bottom right panel.

  8. Click Analyze.
  9. Review the recovered data in the bottom right panel.
  10. Optionally, click View Raw Data to view the individual recorded response data.
  11. If IGNORE appears within any of the recovered questions, compare this to the question from the session file.
  12. Optionally, you can force match any questions that have been ignored. In the Enter forced matches field, enter the recovery question number first and the session question number next, separating the two numbers with a comma. If there are additional forced matches, separate the pair of numbers with a semi-colon.

    You would like to merge recovery question 7 with session question 6 and merge recovery question 8 with session question 7.

    7,6;8,7

  13. Optionally, you can force ignore any matches. In the Enter Recovery to ignore field, enter the recovery question number.
  14. Click Analyze.
  15. When the corresponding questions are set to merge, click Recover.
  16. Next to Recovery Data, click View Report.
  17. Click Save.

    The session is saved as Recovered_Original Session Name.
