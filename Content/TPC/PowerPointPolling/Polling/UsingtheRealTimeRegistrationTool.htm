Using the Real-Time Registration Tool

The Real-Time Registration Tool provides a fast and simple way to assign response devices to a participant list. Registration only takes a few moments, thus eliminating the need for manually adding Device IDs into the permanent participant list. The steps below can be performed before each presentation to ensure each participant has the appropriate response device.

 

A participant list that contains names or User IDs (Device IDs are optional) is required. For instructions on how to create a participant list, see Creating Participant Lists on page 1.

Be sure to make any edits to the participant list prior to using the Real-Time Registration tool.

  1. Click the Participant List drop-down menu and select a participant list.
  2. Click Real-Time Registration on the TurningPoint Cloud ribbontoolbar.

    The Real-Time Registration Tool opens.

    By default 10 participant names display.

    If participants are using clickers with fewer than 10 buttons, such as a ResponseCard LT, click the 10 Participants drop-down menu and select 5 Participants.

  3. Click the Play button to open registration.
  4. Each participant must press a number on their response device that corresponds with the number to the left of their name or User ID.

    The Device ID for the participant will appear in the Device ID column.

  5. Click the forward or back buttons to scroll page by page to see the remaining participants.
  6. Click the Pause button to close registration once all of the participants have responded.
  7. Close the Real-Time Registration Tool.

    The following message appears.

  8. To permanently save the Device IDs to the participant list, click Yes. Otherwise, click No. If you click No, the software will not remember Device IDs for future use.

     

    If No is selected, the participant list is temporarily saved. For information on how to save the Device IDs after the session is completed, see Saving Device IDs from the Real-Time Registration Tool on page 1.

    If the registrations were not saved to the permanent participant list and the session is reset or polling is closed, the Device IDs are not saved.
