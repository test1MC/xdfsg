Sakai® Installation Guide

This guide provides Sakai® System Administrators with step-by-step instructions to install the Turning Technologies tool.

Installing the Tool for Sakai

  1. Deploy the .zip file in the Sakai root directory (under webapps, shared, etc. directories).
  2. (Optional) To add the Turning Technologies Tool icon in the left tool menu in worksites, a small one-time configuration to the Sakai installation is required.

    Identify the skin being used; the following instructions assume the default, but if there is a custom skin you must ensure that the following modifications are made to the skin’s portal.css file.

    1. Open $CATALINA_HOME/webapps/library/skin/default/portal.css

    If there is a skin being used other than the default, switch ‘default’ in that directory with the skin.

    1. Add the following line:

      .icon-sakai-turningtool {background-image: url(/turningtool-tool/img/ttech_icon.ico);}

    1. Save changes and close.
  3. Restart the Sakai Server.
  4. Log in as the Administrator.
  5. Select Realms in the Administration Workspace and select !site.template.course.

  6. Select the Instructor Role ID.
  7. Check all the Turning Tool Functions and click Save under Complete the Realm Edit.

    If using Shibboleth, copy the text below and paste it in
    sakai.properties:#Turning Technologies Shibboleth
    config: turningtool.shibbolethEnabled=true

    If using CAS, copy the text below and paste it in
    sakai.properties:
    #Turning Technologies CAS config: turningtool.casBaseUrl=<CAS_SERVER_PATH>
